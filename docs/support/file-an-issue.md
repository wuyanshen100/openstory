OpenStory is open source, and we track all bugs and feature requests publicly on GitHub. Reporting issues there helps us prioritize fixes and keeps the conversation transparent.

## Open a New Issue

Head to our issue tracker to file a bug or request a feature:

[github.com/openstory-so/openstory/issues/new](https://github.com/openstory-so/openstory/issues/new)

You'll need a free GitHub account to file an issue.

## What to Include in a Bug Report

A good bug report makes the problem easy to reproduce. Please include:

- **What happened** — a clear description of the bug
- **What you expected** — what you thought should have happened
- **Steps to reproduce** — the exact actions that trigger the bug
- **Screenshots or screen recordings** — visuals help enormously
- **Browser and operating system** — e.g. Chrome 121 on macOS 14
- **Console errors** — open your browser's developer tools and copy any red error messages

## Feature Requests

Have an idea for something OpenStory should do? File an issue with the title prefixed `Feature:` and describe:

- The problem you're trying to solve
- How you'd ideally see it work
- Any alternatives you've considered

## Browse Existing Issues

Before filing a new issue, it's worth searching [existing issues](https://github.com/openstory-so/openstory/issues) — your bug or idea may already be tracked. Adding a comment or thumbs-up to an existing issue helps us gauge demand.

## Need Direct Support?

For account-specific questions or anything you'd rather not discuss publicly, [contact us by email](/docs/support/contact-us) instead.
